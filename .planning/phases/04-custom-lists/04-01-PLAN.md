# Phase 4 Plan 01: List Management & Persistence

## Objective
Add ability to save, load, and manage multiple buzzword lists. Users can choose which list to play with.

## Context
@Buzzword Bingo/Buzzword Bingo/Models/BuzzwordList.swift
@Buzzword Bingo/Buzzword Bingo/Models/GameSession.swift
@Buzzword Bingo/Buzzword Bingo/Views/PlayerSetupView.swift

## Tasks

### Task 1: Create ListStore for Persistence
**Type:** create
**Files:** `Buzzword Bingo/Buzzword Bingo/Models/ListStore.swift`
**Action:**
- @Observable class ListStore
- Properties: `lists: [BuzzwordList]`, `selectedListId: UUID?`
- Load/save to JSON file in app documents directory
- Auto-include default list (can't be deleted)
- Methods: `save()`, `load()`, `add(list:)`, `delete(id:)`, `update(list:)`

**Verify:** File compiles

### Task 2: Create ListPickerView
**Type:** create
**Files:** `Buzzword Bingo/Buzzword Bingo/Views/ListPickerView.swift`
**Action:**
- Show all available buzzword lists
- Tap to select for upcoming game
- Swipe to delete (except default)
- "Create New" button
- Show word count per list

**Verify:** Preview renders

### Task 3: Integrate List Selection into PlayerSetupView
**Type:** modify
**Files:** `Buzzword Bingo/Buzzword Bingo/Views/PlayerSetupView.swift`
**Action:**
- Add list picker before player names
- Pass selected BuzzwordList to onStart callback
- Update callback signature: `onStart: ([String], BuzzwordList) -> Void`

**Verify:** Builds pass

### Task 4: Update ContentView and GameSession
**Type:** modify
**Files:** `Buzzword Bingo/Buzzword Bingo/ContentView.swift`
**Action:**
- Create ListStore as @State
- Pass selected list through game flow
- Update startGame to use selected list

**Verify:** Both platforms build

## Verification
- [ ] ListStore persists data between app launches
- [ ] Default list always available
- [ ] Can select different list for game
- [ ] Builds pass on both platforms

## Success Criteria
- Lists persist to disk
- Default list cannot be deleted
- Game uses selected list for card generation
